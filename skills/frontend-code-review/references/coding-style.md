# 编码风格规范

## 一、缩进

- **【强制】** 使用 2 个空格作为一个缩进层级，不允许使用 4 个空格或 tab 字符。新项目需强制遵循，存量项目可酌情决策。落地方式：静态代码扫描。

## 二、特殊符号

### 1. 分号

- **【强制】** 强制语句末尾使用分号。新项目需强制遵循，存量项目可酌情决策。落地方式：静态代码扫描。
  - 语句体（函数、if/while/for 等控制流）的右括号 `}` 后不需要加分号，但箭头函数需要。
  - 如果函数定义被当作变量赋值给一个变量，该函数定义右括号 `}` 后需要加分号。
- **【强制】** 不允许出现不必要的分号。落地方式：静态代码扫描。

### 2. 逗号

- **【强制】** 禁止将逗号置于行首，逗号应该紧跟上个变量、字符串等临边源码块。落地方式：静态代码扫描。
- **【强制】** 添加额外结尾逗号（trailing comma）——对象属性、枚举声明等使用逗号场景，最后一项也需要添加逗号。好处是增删时 Git diff 不会动不涉及的代码行。落地方式：静态代码扫描。
- **【强制】** 如果单行一段源码块以 `]`、`)` 或 `}` 结尾，最后一项的逗号不需要添加；对象属性、枚举声明等不鼓励单行声明。落地方式：静态代码扫描。
- **【强制】** 逗号处在一行非最后一个字符时需在后面空一格。落地方式：静态代码扫描。

### 3. 引号

- **【强制】** 默认强制使用单引号 `'`，以下两种场景接受双引号：
  - ❶ 字符串中内容含有单引号字符时，可以接受双引号。
  - ❷ JSX 或 Vue Template 那种 HTML 标签风格属性要求使用双引号。
  - 落地方式：静态代码扫描。

### 4. 空格和换行

- **【强制】** 在函数、类、枚举或结构体等声明后面出现的左大括号 `{` 前需空一格。落地方式：静态代码扫描。
- **【强制】** 在控制语句（if、while 等）的条件前后需空一格。落地方式：静态代码扫描。
- **【强制】** 在二元运算符和三元运算符的两侧各空一格。落地方式：静态代码扫描。
- **【强制】** 在注释内容前空一格。落地方式：静态代码扫描。
- **【强制】** 禁止行尾出现多余空格。落地方式：静态代码扫描。
- **【强制】** 禁止出现多个连续空行，最多允许一个空行。落地方式：静态代码扫描。
- **【强制】** 单行代码最大长度不超过 120 个字符。落地方式：静态代码扫描。
- **【强制】** 文件末尾保留一个空行。落地方式：静态代码扫描。
- **【建议】** 函数声明、赋值语句、return 语句前后保留空行以增强可读性。落地方式：CodeReview。

### 5. 括号

- **【强制】** 使用大括号包裹所有的多行代码块。落地方式：静态代码扫描。
- **【强制】** 如果通过 if 和 else 使用多行代码块，把 else 放在 if 代码块闭合括号的同一行。落地方式：静态代码扫描。

## 三、注释

### 1. 规范

- **【强制】** 多行注释使用 `/** ... */`。落地方式：静态代码扫描。
- **【强制】** 单行注释使用 `//`，将单行注释放在被注释区域上方独立一行，在注释前插入一个空行（除非注释在块的第一行）。落地方式：静态代码扫描。
- **【强制】** 所有注释开头空一格，方便阅读。落地方式：静态代码扫描。
- **【建议】** 使用 `// FIXME:` 标注问题，使用 `// TODO:` 标注待办。落地方式：CodeReview。

### 2. 注释标记

| 标记 | 用途 |
|------|------|
| TODO | 表示后面部分源码并没有完成所有功能，后续需要完善。 |
| FIXME | 表示后续源码虽然能够运行，但存在可预期的缺陷或不稳定场景。 |
| HACK | 指该功能的实现使用的是临时方案，或实现并不优雅。 |
| NOTE | 用于强调代码中的某个特别重要的信息。 |
| OPTIMIZE | 提示代码的性能可以进一步优化。 |
| REVIEW | 表示代码需要额外的审查。 |
| DEPRECATED | 表明代码已经过时，不应该被使用或需要重构。 |
| NOCOMMIT | 警告代码不应该被提交到版本控制系统中，常用于临时调试代码。 |

## 四、命名

### 1. 有效命名字符集

- **【强制】** 源码中的命名只能包含英文字母（A-Za-z）、数字（0-9）、下划线（_）和美元符号（$）。禁止使用中文、特殊字符。落地方式：静态代码扫描。

### 2. 文件命名

- **【强制】** 普通文件和目录使用连字符命名法（kebab-case）。落地方式：静态代码扫描。
- **【强制】** React 组件文件使用帕斯卡命名法（PascalCase）。落地方式：静态代码扫描。
- **【强制】** 测试文件以 `.test.js`、`.spec.js`、`.test.ts`、`.spec.ts` 结尾。落地方式：静态代码扫描。

### 3. 源码命名

- **【强制】** 变量和函数使用小驼峰命名法（camelCase）。落地方式：静态代码扫描。
- **【强制】** 类和构造函数使用帕斯卡命名法（PascalCase）。落地方式：静态代码扫描。
- **【强制】** 常量使用宏命名法（UPPER_SNAKE_CASE）。落地方式：静态代码扫描。
- **【强制】** 布尔类型变量使用 is、has、can、should 等前缀。落地方式：CodeReview。
- **【强制】** 枚举类型使用帕斯卡命名法。落地方式：静态代码扫描。
- **【建议】** 避免使用单字符命名（循环变量 i/j/k 除外）。落地方式：CodeReview。
- **【建议】** 避免使用缩写，除非是行业公认的标准缩写。落地方式：CodeReview。

### 4. 专有名词首字母大写规则

| 专有名词 | 正确用法 | 错误用法 |
|----------|----------|----------|
| ID | getID()、const userID = 231480 | getId()、const userId = 231480 |
| DOM | getDOMNode() | getDomNode() |
| HTTP | sendHTTPRequest() | sendHttpRequest() |
| iOS/macOS/iPadOS | getMacOSVersion() | getMACOSVersion() |
| OAuth | setOAuthConfig() | setOauthConfig() |
| MySQL/SQLite | getMySQLHostName() | getMysqlHostName() |

## 五、其他

### 1. 函数

- **【强制】** 函数参数不超过 5 个，超过时使用对象参数。落地方式：静态代码扫描。
- **【强制】** 函数体不超过 80 行（不含空行和注释）。落地方式：静态代码扫描。
- **【建议】** 函数应该只做一件事（单一职责原则）。落地方式：CodeReview。
- **【建议】** 函数嵌套层级不超过 4 层。落地方式：静态代码扫描。

### 2. 复杂度

- **【强制】** 圈复杂度不超过 20。落地方式：静态代码扫描。
- **【建议】** 圈复杂度不超过 10。落地方式：CodeReview。

### 3. 健壮性

- **【强制】** 对外部输入数据进行校验。落地方式：CodeReview。
- **【强制】** 对可能为 null/undefined 的值进行判空处理。落地方式：CodeReview。
