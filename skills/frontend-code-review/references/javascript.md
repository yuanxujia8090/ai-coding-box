# JavaScript 编码规范

## 一、变量

- **【强制】** 使用 const 或 let 声明局部变量。变量需要改变引用时使用 let，否则使用 const。禁止使用 var。落地方式：静态代码扫描。
- **【强制】** 一次声明一个变量。禁止一次声明多个变量。禁止链式声明。落地方式：静态代码扫描。
- **【强制】** 连续声明多种变量时，把使用 const 或 let 声明的变量分别放在一起。落地方式：CodeReview。
- **【强制】** 禁止声明不被使用的变量。落地方式：静态代码扫描。
- **【强制】** 禁止声明外层作用域已有变量名（变量遮蔽）。落地方式：静态代码扫描。
- **【建议】** 尽量在贴近变量首次使用的位置前声明变量，避免在块级结构的开始声明块中使用的所有变量。落地方式：CodeReview。

## 二、对象

- **【强制】** 使用字面值创建对象，不要使用 Object 构造器。落地方式：静态代码扫描。
- **【强制】** 使用对象方法的简写形式。落地方式：静态代码扫描。
- **【强制】** 优先使用简写属性值。落地方式：静态代码扫描。
- **【强制】** 仅对无效标识符的属性使用引号，其他情况禁止在属性上使用引号。落地方式：静态代码扫描。
- **【强制】** 禁止直接调用 Object.prototype 上的方法，如 hasOwnProperty、propertyIsEnumerable、isPrototypeOf 等。这些方法可能会被对象自身的同名属性覆盖，或者对象是 null（Object.create(null)）。落地方式：静态代码扫描。
- **【强制】** 使用对象展开语法（...）而不是 Object.assign 实现对象的浅拷贝。使用对象其余属性语法获取一个去除指定属性的新对象。落地方式：静态代码扫描。

## 三、数组

- **【强制】** 使用字面量语法创建数组，禁止使用 Array 构造函数（创建指定大小的稀疏数组除外）。落地方式：静态代码扫描。
- **【强制】** 使用展开运算符 `...` 做数组浅拷贝。落地方式：静态代码扫描。
- **【强制】** 使用 Array.from 或展开运算符将类数组对象转为数组。落地方式：静态代码扫描。
- **【强制】** 在数组方法的回调函数中使用 return 语句。如果函数体由一条返回无副作用的表达式的语句组成，可以省略 return。落地方式：静态代码扫描。
- **【建议】** 如果一个数组有很多行，在数组的 `[` 后和 `]` 前断行。落地方式：CodeReview。

## 四、解构

- **【强制】** 使用解构存取和使用多属性对象。落地方式：静态代码扫描。
- **【强制】** 对数组使用解构赋值。落地方式：静态代码扫描。
- **【建议】** 函数需要回传多个值时，使用对象解构而不是数组解构。落地方式：CodeReview。

## 五、字符串

- **【强制】** 使用单引号 `''` 定义字符串。落地方式：静态代码扫描。
- **【强制】** 禁止使用不必要的转义字符。落地方式：静态代码扫描。
- **【强制】** 使用模板字符串而非字符串拼接来组织可编程字符串。落地方式：静态代码扫描。
- **【强制】** 禁止对字符串使用 eval()。落地方式：静态代码扫描。

## 六、函数

- **【强制】** 使用函数声明代替函数表达式。函数声明是可命名的，在调用栈中更容易被识别，且会被提升（hoisted）。落地方式：CodeReview。
- **【强制】** 禁止在非函数代码块（if、while 等）中声明函数。落地方式：静态代码扫描。
- **【强制】** 禁止使用 arguments，用剩余参数操作符 `...` 替代。落地方式：静态代码扫描。
- **【强制】** 使用默认参数语法，而不是修改函数参数。落地方式：静态代码扫描。
- **【强制】** 默认参数放在最后。落地方式：CodeReview。
- **【强制】** 禁止修改参数（mutate parameters）。落地方式：静态代码扫描。
- **【强制】** 禁止对参数重新赋值。落地方式：静态代码扫描。

## 七、箭头函数

- **【强制】** 当必须使用匿名函数时（如传递内联回调），使用箭头函数。落地方式：静态代码扫描。
- **【强制】** 如果函数体由一个没有副作用的表达式语句组成，删除大括号和 return，使用隐式返回。否则保留大括号和 return。落地方式：CodeReview。
- **【强制】** 如果表达式涉及多行，用圆括号包裹以提高可读性。落地方式：CodeReview。
- **【强制】** 如果函数只有一个参数且不使用大括号，省略圆括号。否则始终在参数周围使用圆括号。落地方式：静态代码扫描。
- **【强制】** 避免箭头函数语法 `=>` 与比较运算符 `<=`、`>=` 混淆。落地方式：CodeReview。

## 八、类与构造函数

- **【强制】** 使用 class 语法，避免直接操作 prototype。落地方式：CodeReview。
- **【强制】** 使用 extends 实现继承。落地方式：CodeReview。
- **【强制】** 方法可以返回 this 来实现链式调用。落地方式：CodeReview。
- **【强制】** 禁止出现重复的类成员。落地方式：静态代码扫描。
- **【建议】** 如果没有指定类，类有一个默认的构造函数，不需要写一个空的构造函数或只是代表父类的构造函数。落地方式：CodeReview。

## 九、模块

- **【强制】** 使用 ES Module（import/export）而非其他非标准模块系统。落地方式：静态代码扫描。
- **【强制】** 禁止使用通配符 import（`import *`）。落地方式：静态代码扫描。
- **【强制】** 禁止直接从 import 中 export。落地方式：CodeReview。
- **【强制】** 一个路径只 import 一次。落地方式：静态代码扫描。
- **【强制】** 禁止导出可变绑定（export let）。落地方式：静态代码扫描。
- **【强制】** 在只有一个导出的模块中，使用 default export。落地方式：CodeReview。
- **【强制】** 将所有 import 语句放在非 import 语句之上。落地方式：静态代码扫描。
- **【建议】** 多行 import 应该像多行数组和对象字面量一样缩进。落地方式：CodeReview。
- **【建议】** 禁止在 import 语句中使用 Webpack loader 语法。落地方式：静态代码扫描。

## 十、迭代器与生成器

- **【强制】** 禁止使用迭代器（for-in、for-of），优先使用 JavaScript 的高阶函数（map、every、filter、find、findIndex、reduce 等）代替 for-in 或 for-of 循环。落地方式：CodeReview。

## 十一、属性

- **【强制】** 使用点号 `.` 访问属性。落地方式：静态代码扫描。
- **【强制】** 当通过变量访问属性时使用中括号 `[]`。落地方式：静态代码扫描。

## 十二、运算符

- **【强制】** 使用 `===` 和 `!==` 而不是 `==` 和 `!=`。落地方式：静态代码扫描。
- **【强制】** 三元表达式不应该嵌套，通常是单行表达式。落地方式：静态代码扫描。
- **【强制】** 禁止不必要的三元表达式。落地方式：静态代码扫描。
- **【强制】** 混合使用不同运算符时，用小括号括起来明确优先级。`**` 和 `%` 除外。落地方式：静态代码扫描。
- **【强制】** 禁止使用一元操作符 `++` 和 `--`，使用 `num += 1` 代替 `num++`。落地方式：静态代码扫描。
- **【强制】** 避免不必要的条件表达式。落地方式：CodeReview。

## 十三、块

- **【强制】** 使用大括号包裹所有的多行代码块。落地方式：静态代码扫描。
- **【强制】** 如果通过 if 和 else 使用多行代码块，把 else 放在 if 代码块闭合括号的同一行。落地方式：静态代码扫描。
- **【强制】** 如果 if 块中包含 return 语句，else 块就不是必须的，可以将其内容移到 if 块后面。落地方式：CodeReview。

## 十四、控制语句

- **【强制】** 当控制语句（if、while 等）太长或超过最大行长度时，每个条件放在单独一行，逻辑运算符放在行首。落地方式：CodeReview。
- **【强制】** 禁止使用选择操作符代替控制语句。落地方式：CodeReview。

## 十五、类型和类型转换

- **【强制】** 在语句开始时执行类型转换。落地方式：CodeReview。
- **【强制】** 字符串转换使用 String()。落地方式：CodeReview。
- **【强制】** 数字转换使用 Number() 或 parseInt()（始终带上基数）。落地方式：静态代码扫描。
- **【强制】** 布尔转换使用 Boolean() 或 `!!`。落地方式：CodeReview。

## 十六、属性访问函数

- **【建议】** 不要使用 JavaScript 的 getters/setters，因为它们会产生副作用且难以测试、维护和理解。如果需要存取函数，使用 getVal() 和 setVal('hello')。落地方式：CodeReview。

## 十七、事件

- **【建议】** 当给事件附加数据时，传入一个对象而不是原始值。这样在后续的事件处理中可以方便地添加更多数据，而不需要找到并更新每一个处理器。落地方式：CodeReview。

## 十八、标准库

- **【强制】** 使用 Number.isNaN 代替全局的 isNaN。全局 isNaN 会将非数字强制转为数字，任何被强制转为 NaN 的东西都会返回 true。落地方式：静态代码扫描。
- **【强制】** 使用 Number.isFinite 代替全局的 isFinite。全局 isFinite 会将非数字强制转为数字，任何被强制转为有限数字的东西都会返回 true。落地方式：静态代码扫描。

## 十九、Promise

- **【强制】** 优先使用 async/await 而非 Promise 链式调用。落地方式：CodeReview。
- **【强制】** Promise 必须有 catch 处理或在 async 函数中使用 try/catch。落地方式：静态代码扫描。
