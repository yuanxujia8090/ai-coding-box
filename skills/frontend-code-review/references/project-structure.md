# 文件、目录和包规范

## 一、命名

### 命名法定义

| 命名法 | 要求 | 举例 |
|--------|------|------|
| 连字符命名法 | 只能包含英文小写字母(a-z)、连字符(-)和数字(0-9)。禁止首位字符出现数字。单词间以连字符分隔。 | user-profile、nav-bar |
| 驼峰命名法 | 只能包含英文大小写字母(A-Za-z)和数字(0-9)。禁止首位字符出现数字。第一个单词首字母可小写，从第二个单词开始首字母大写。 | 小驼峰：firstName、userProfile；大驼峰：FirstName、UserProfile |
| 帕斯卡命名法 | 只能包含英文大小写字母(A-Za-z)和数字(0-9)。禁止首位字符出现数字。每个单词首字母大写。 | Car、LinkedList |
| 蛇形命名法 | 只能包含英文小写字母(a-z)、下划线(_)和数字(0-9)。禁止首位字符出现数字。单词间以下划线分隔。 | first_name、user_profile |
| 宏命名法 | 只能包含英文大写字母(A-Z)、下划线(_)和数字(0-9)。禁止首位字符出现数字。单词间以下划线分隔。 | MAX_SIZE |

## 二、文件规范

### 4.1 命名

- **【强制】** 文件名使用连字符命名法。
- **【强制】** React 组件文件使用帕斯卡命名法，扩展名为 .jsx 或 .tsx。
- **【强制】** 测试文件后缀为 `.test.js`、`.spec.js`、`.test.ts`、`.spec.ts`。

### 4.2 文件

- **【强制】** 文件编码使用 UTF-8。
- **【强制】** 文件末尾保留一个空行。
- **【强制】** 每个文件只包含一个组件（无 state 的纯组件除外）。

### 4.3 目录

**【强制】** 目录名使用连字符命名法。

常用目录命名约定：

| 目录名 | 说明 |
|--------|------|
| img | 图片。不使用 image、images、imgs 等。 |
| js | javascript 脚本。不使用 script、scripts 等。 |
| css | 样式表。不使用 style、styles 等。 |
| deps | 引入的第三方依赖包目录。 |
| docs | 文档目录。 |
| typings | TypeScript 类型目录。 |
| __tests__ | 测试用例目录。 |
| components | 通用组件、或纯组件，跟业务绑定浅。 |
| utils | 通用的工具函数，建议一个项目保留一个 utils 目录，保证纯函数实现都在一处，避免重复。 |
| helpers | 业务逻辑抽象，与 utils 的区别是：❶ utils 面向整个项目，helpers 面向具体业务；❷ 一个项目原则上只有一个 utils，但 helpers 目录会散落在业务模块中。 |
| views | view 相关组件/页面。 |
| pages | 页面相关的目录。 |
| modules | 具体业务模块组件，一般跟 Stores、React Context 耦合。 |

### 4.4 包

- **【强制】** 使用 package.json 管理项目依赖。
- **【强制】** 使用 lockfile（package-lock.json 或 yarn.lock）锁定依赖版本。
- **【建议】** 定期更新依赖，及时修复安全漏洞。
